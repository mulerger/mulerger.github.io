Technical refresher

Git and GitHub

A practical workflow for collaborative analysis, documentation and prototypes

Dr Mesfin Diro · Technical Facilitator

The difference

Git records the work; GitHub connects the family

Git

Version-control software on your computer. It records who changed what, when and why.

GitHub

The shared online repository for branches, issues, pull requests, review and published project history.

A commit is a meaningful snapshot—not a copy named “final-final-new”.

First setup

Identify yourself and clone the family repository

git config --global user.name "Your Name"
git config --global user.email "you@example.org"
git clone <family-repository-url>
cd <family-repository>

Use the name and email linked to your GitHub account so your contributions are recorded correctly.

Daily workflow

Pull, branch, edit, commit and push

git pull
git switch -c analysis/short-task-name

# edit and check your files
git add path/to/file
git commit -m "Explain the change"
git push -u origin analysis/short-task-name

Create a focused branch for one task. Pull before starting so you build on the latest shared work.

Repository structure

Keep every output in a predictable place

manuscript/

Manuscript-style report

code/

Preparation, modelling and validation scripts

data/

Dictionary, access notes and approved shareable data

documentation/

Methods, assumptions and guidance

prototype/

Dashboard or working application

outputs/

Tables, maps and figures

slides/

Final family presentation

Pull requests

Review before merging

Open

State the task, what changed and how the result was checked.

Review

Read the code and output; ask questions on the relevant lines.

Merge

Resolve comments, confirm checks and merge the completed work.

Good collaboration

Make the history easy to understand

  • Use a short branch name that describes one task.
  • Commit related changes together with a clear action message.
  • Use issues to assign work, record decisions and track blockers.
  • Update the README when setup, inputs or outputs change.
  • Pull and review before merging to reduce avoidable conflicts.
