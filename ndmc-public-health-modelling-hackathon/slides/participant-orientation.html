Family Workflow, Git and GitHub

A practical workflow for multidisciplinary analysis, documentation and prototypes

Dr Mesfin Diro · Technical Facilitator

Family workflow

Form, align, build and review

1 · Form

Bring together programme, modelling, data and communication experience.

2 · Align

Agree on the question, roles, tasks and the day's visible output.

3 · Build

Work in parallel on data, methods, code, interpretation and documentation.

4 · Review

Demonstrate progress, resolve blockers and define the next step.

Family roles

Clear responsibility, shared ownership

Facilitator

Keeps the family focused, connects support and helps resolve blockers.

GitHub coordinator

Maintains repository structure, branches, pull requests and merge readiness.

Modelling and data contributors

Develop the analytical workflow, test assumptions and document results.

Policy liaison and presenter

Connects findings to the decision question and leads the final demonstration.

Daily rhythm

Make progress visible every day

Start

Agree priorities, owners and the day's checkpoint.

Work

Use focused blocks for data, modelling, code and writing.

Check

Review the method and policy interpretation with programme leads.

Close

Merge completed work, record blockers and set tomorrow's first task.

The difference

Git records the work; GitHub connects the family

Git

Version-control software that records who changed what, when and why.

GitHub

The shared online repository for branches, issues, pull requests and review.

A commit is a meaningful snapshot of the work and the reason it changed.

First setup

Identify yourself and clone the family repository

git config --global user.name "Your Name"
git config --global user.email "you@example.org"
git clone <family-repository-url>
cd <family-repository>

Use the name and email linked to your GitHub account so your contributions are recorded correctly.

Daily Git workflow

Pull, branch, edit, commit and push

git pull
git switch -c analysis/short-task-name

# edit and check your files
git add path/to/file
git commit -m "Explain the change"
git push -u origin analysis/short-task-name

Use one focused branch for one task and pull the latest shared work before starting.

Repository structure

Keep every output in a predictable place

manuscript/

Manuscript-style report

code/

Preparation, modelling and validation scripts

data/

Dictionary, access notes and approved shareable data

documentation/

Methods, assumptions and guidance

prototype/

Dashboard or working application

outputs/

Tables, maps and figures

slides/

Final family presentation

Pull requests

Review before merging

Open

State the task, what changed and how it was checked.

Review

Read the code and output; ask questions on the relevant lines.

Merge

Resolve comments, confirm checks and merge the completed work.

Communication

Use the right place for each conversation

Family update

Use the agreed family channel for priorities, short updates and meeting notes.

Code review

Keep technical discussion with the relevant issue or pull request on GitHub.

Blocker

Bring a focused question, the attempted solution and the decision needed.
